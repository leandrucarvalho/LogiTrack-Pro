Desafio Técnico: LogiTrack Pro 1. Contexto A LogiTrack é uma empresa fictícia do setor de logística e transportes que opera frotas para entregas interestaduais. Atualmente, a empresa utiliza planilhas isoladas para controlar seus veículos, o que gera erros de agendamento, falta de visibilidade sobre os custos de manutenção e baixa previsibilidade na produtividade da frota.  Para resolver isso, você foi selecionado para desenvolver o MVP (Mínimo Produto Viável) de um sistema web que centralize essas informações e forneça inteligência de dados para os gestores da empresa.  2. Desafio Seu objetivo é criar uma aplicação web utilizando Spring Boot que permita o gerenciamento de uma das operações principais e apresente um painel de análise de dados (Dashboard) baseado em informações do banco de dados fornecido. Requisitos Funcionais A. Gestão de Dados (CRUD) Você deve escolher e implementar UM dos módulos abaixo: ● Opção 1: Módulo de Viagens ○ Campos: Seleção de Veículo, Data/Hora de Saída, Data/Hora de Chegada, Cidade de Origem, Cidade de Destino e Quilometragem Percorrida. ● Opção 2: Módulo de Agendamento de Manutenção ○ Campos: Seleção de Veículo, Data de Início, Data de Finalização prevista, Tipo de Serviço (ex: Troca de Pneus, Motor, Óleo), Custo Estimado e Status (ex: Pendente, Em Realização, Concluída). B. Dashboard de Análise Independente do módulo de CRUD escolhido, o sistema deve exibir uma área de relatórios com as seguintes 5 métricas extraídas obrigatoriamente via SQL: ● Total de KM percorrido: Soma da quilometragem de um veículo específico ou de toda a frota. ● Volume por Categoria: Quantidade de viagens realizadas filtradas por tipo de veículo (Leve vs Pesado).

● Cronograma de Manutenção: Listagem das próximas 5 manutenções agendadas (ordenadas por data). ● Ranking de Utilização: Identificar qual veículo possui a maior soma de quilometragem acumulada. ● Projeção Financeira: Soma do custo total estimado em manutenções para o mês atual. 3. Requisitos Técnicos ● Backend: Uso de Spring Boot com o Java versão 17 ou superior. ● Banco de Dados: Utilizar o script SQL fornecido em anexo. Você tem liberdade total para adicionar novas tabelas ou colunas caso sinta necessidade. ● Frontend: Interface web funcional. Pode ser desenvolvida com templates (Thymeleaf/JSF) ou SPAs. Diferenciais: ● Frontend Moderno: Uso de frameworks Javascript (React, Vue ou Angular). ● Segurança: Implementação de tela de login e autenticação de usuários. ● DevOps: Disponibilizar o ambiente via Docker ou realizar o deploy em um serviço de nuvem (Render, Railway, AWS, etc). 4. Instruções para Entrega O projeto deve ser entregue via repositório público no GitHub ou GitLab e deve conter:  ● Código-fonte: Organizado, limpo e seguindo boas práticas de POO. ● README.md: Arquivo contendo: ○ Instruções claras de como configurar e rodar o projeto localmente. ○ Descrição das decisões técnicas, ferramentas e arquitetura escolhida. ○ Caso tenha alterado o banco de dados, explique as justificativas e inclua o novo script. ● Demonstração: Link do deploy (se houver) ou prints das funcionalidades principais.  O que avaliaremos: Qualidade da modelagem, organização de camadas no Java, eficiência das consultas SQL e clareza na documentação.   Boa sorte!
